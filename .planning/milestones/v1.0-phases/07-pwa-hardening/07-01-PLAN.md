<task type="auto">
  <name>Task 1: Fix Workbox caching — reduce Mapbox maxEntries, add Supabase NetworkFirst</name>
  <files>vite.config.ts</files>
  <action>
Modify the existing `runtimeCaching` array in `vite.config.ts`:

1. **Reduce Mapbox tile cache maxEntries from 500 to 50.** The existing entry at `urlPattern: /^https:\/\/api\.mapbox\.com\/.*/i` has `maxEntries: 500`. Change to `maxEntries: 50`. This prevents opaque response quota exhaustion (500 * ~7 MB = 3.5 GB theoretical quota attribution vs 50 * ~7 MB = 350 MB — safe). Keep all other options unchanged (CacheFirst, cacheName, maxAgeSeconds, purgeOnQuotaError, cacheableResponse).

2. **Add a new Supabase NetworkFirst runtime cache entry** after the Mapbox entry:
```typescript
{
  urlPattern: /^https:\/\/[a-z0-9]+\.supabase\.co\/.*/i,
  handler: 'NetworkFirst',
  options: {
    cacheName: 'psi-szlak-supabase-trails',
    expiration: {
      maxEntries: 10,
      maxAgeSeconds: 60 * 60 * 24 * 7, // 7 days
      purgeOnQuotaError: true,
    },
    cacheableResponse: { statuses: [0, 200] },
    networkTimeoutSeconds: 3, // fall back to cache after 3s
  },
},
```

Do NOT change anything else in vite.config.ts (plugins, globPatterns, maximumFileSizeToCacheInBytes).

  </verify>
  <done>Mapbox maxEntries is 50 (not 500). Supabase NetworkFirst cache entry exists with cacheName psi-szlak-supabase-trails, maxEntries 10, networkTimeoutSeconds 3. Build succeeds.</done>

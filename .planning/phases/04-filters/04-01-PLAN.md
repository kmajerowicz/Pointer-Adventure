<feature>
  <name>useFilteredRoutes — client-side trail filtering</name>
  <files>src/hooks/useFilteredRoutes.ts, src/hooks/useFilteredRoutes.test.ts</files>
  <behavior>
    - Length 'short': includes routes with length_km < 5 or null; excludes routes >= 5
    - Length 'medium': includes routes with 5 <= length_km <= 15 or null; excludes outside range
    - Length 'long': includes routes with length_km > 15 or null; excludes <= 15
    - Length null: no filtering (all pass)
    - Surface 'dirt': only routes with surface_type === 'dirt'; null means no filtering
    - Difficulty 'moderate': only routes with difficulty === 'moderate'; null means no filtering
    - Water 'required': excludes routes with water_access === 'none'
    - Water 'preferred': keeps all routes but sorts water_access !== 'none' to top (stable sort)
    - Water 'any': no filtering or sorting
    - Marked true: excludes routes with is_marked === false; null means no filtering
    - Distance 10: excludes routes > 10km from user GPS position; skipped if GPS unavailable
    - Distance null: no distance filtering
    - Multiple filters combine (AND logic): length short + surface dirt = only short dirt trails
    - No Edge Function call when filters change (verified by absence of fetch mock)
  </behavior>
  <implementation>
    Create useFilteredRoutes hook using useMemo over useTrailsStore.routes + useFiltersStore state + useGeolocation position.
    Filter chain: length -> surface -> difficulty -> water required -> marked -> distance.
    Water 'preferred' sort applied after all filters using [...result].sort() (never mutate).
    Skip distance filter when userLat/userLon are null (geolocation not available).
    Routes with null length_km pass through length filter (include unknown lengths).
  </implementation>
</feature>

In `src/features/map/LocationSearch.tsx`, make these changes:

1. **Remove the useEffect at lines 31-40** that wires `map.on('dragstart', removeMarker)`. This effect is broken because `mapRef.current` is null at mount time and the stable ref identity never triggers a re-run.

2. **Inside the `handleSelect` function**, after the marker is placed (after `markerRef.current = new mapboxgl.Marker(...).setLngLat(...).addTo(map)`), add:
   ```typescript
   // Remove marker on next user pan — once() self-cleans after firing
   map.once('dragstart', removeMarker)
   ```

3. **In the `handleClear` function**, before or after `removeMarker()`, also detach any pending dragstart listener to prevent it from firing after the marker is already cleared:
   ```typescript
   const map = mapRef.current
   if (map) {
     map.off('dragstart', removeMarker)
   }
   ```

4. **In the cleanup effect** (the one at lines 58-64 that cleans up debounce timer), also clean up the dragstart listener to prevent it from firing after unmount:
   ```typescript
   const map = mapRef.current
   if (map) {
     map.off('dragstart', removeMarker)
   }
   ```

Why `map.once` instead of `map.on`: The `once` variant fires exactly once per marker placement, matching the one-per-marker behavior. It self-cleans after firing, so no manual `off()` is needed for the normal flow. The `off()` in handleClear and cleanup is defensive — it handles the case where the user clears search or unmounts before panning.

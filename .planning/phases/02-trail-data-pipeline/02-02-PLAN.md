    **1. Create `src/stores/trails.ts`** — Zustand store following the same pattern as viewport.ts:
    ```
    interface TrailsState {
      routes: Route[]
      loading: boolean
      error: string | null
      lastFetched: string | null  // ISO timestamp for cache display
      setRoutes: (routes: Route[]) => void
      appendRoutes: (routes: Route[]) => void  // accumulate pins from new areas
      setLoading: (loading: boolean) => void
      setError: (error: string | null) => void
      setLastFetched: (ts: string | null) => void
      retry: (() => void) | null
      setRetry: (fn: (() => void) | null) => void
    }
    ```
    Use `appendRoutes` to MERGE new routes with existing ones (deduplicate by `id` or `source_id`). This implements the "pins accumulate as user explores" decision. `setRoutes` replaces all routes (used for forced refresh).

    **2. Create `src/hooks/useTrails.ts`**:
    - Subscribe to `useViewportStore((s) => s.bounds)`.
    - On bounds change, set a 400ms debounce timer (useEffect + setTimeout + clearTimeout pattern).
    - Inside the debounce callback: use a `boundsRef` (useRef updated on every render) to read the LATEST bounds, avoiding the stale closure pitfall from research Pitfall 4.
    - Call `supabase.functions.invoke('search-trails', { body: { north, south, east, west } })`.
    - On success: call `appendRoutes(data.routes)` and `setLastFetched(new Date().toISOString())`.
    - On error: call `setError('Nie udalo sie pobrac tras')` and `setRetry(() => retryFn)` where retryFn re-invokes with the same bounds.
    - Set loading true before fetch, false in finally block.
    - Export a `forceRefresh()` function that calls the Edge Function with `{ ...bounds, force: true }` and uses `setRoutes` (replace, not append) for the current viewport.
    - Hook returns `{ loading, error, retry, forceRefresh }` for UI consumption.

    **3. Create `src/features/map/TrailLayers.ts`** — Pure functions (no React, no hooks) that operate on a mapboxgl.Map instance:

    `initTrailLayers(map: mapboxgl.Map)`:
    - Guard with `if (map.getSource('trails'))` to handle StrictMode double-init (Pitfall 3).
    - Add GeoJSON source `'trails'` with `cluster: true, clusterMaxZoom: 14, clusterRadius: 50` and empty FeatureCollection.
    - Add circle layer `'trail-clusters'` filtered by `['has', 'point_count']` — accent gold (#C9A84C) circles with step-based radius (16/22/28 for 10/50+ counts), dark stroke. Per user decision: minimal colored dots.
    - Add symbol layer `'trail-cluster-count'` with `text-field: ['get', 'point_count_abbreviated']`, dark text color.
    - Add circle layer `'trail-unclustered'` filtered by `['!', ['has', 'point_count']]` — use `match` expression on `trail_color` property: red=#C0392B, blue=#2980B9, yellow=#F1C40F, green=#27AE60, black=#2C3E50, default=#C9A84C (accent gold for non-PTTK). Circle radius 6, stroke 1.5px dark.
    - Add GeoJSON source `'trails-lines'` with empty FeatureCollection (no clustering).
    - Add line layer `'trail-line-casing'` — dark (#1a1a1a), width 7px, round join/cap. Per user decision: dark outline casing.
    - Add line layer `'trail-line-fill'` — color from `match` on `trail_color` (same colors as dots), width 4px, round join/cap. Per user decision: 4-5px medium weight.

    `updateTrailData(map: mapboxgl.Map, routes: Route[])`:
    - Build point FeatureCollection from ALL routes (center_lat/center_lon as Point geometry, properties: id, name, trail_color, source).
    - Build line FeatureCollection from PTTK routes with geometry (filter source==='pttk' && geometry exists, use geometry as-is, properties: id, name, trail_color). Per user decision: show BOTH pin and polyline for PTTK.
    - Call `(map.getSource('trails') as GeoJSONSource).setData(pointFC)`.
    - Call `(map.getSource('trails-lines') as GeoJSONSource).setData(lineFC)`.

    `setupTrailInteractions(map: mapboxgl.Map)`:
    - On click `'trail-clusters'`: get cluster expansion zoom via `(map.getSource('trails') as GeoJSONSource).getClusterExpansionZoom(clusterId)`, then `map.easeTo({ center, zoom })`.
    - On click `'trail-unclustered'`: show a Mapbox Popup with the trail name from feature properties. Per user decision: tapping pin shows small popup with trail name.
    - On click `'trail-line-fill'`: show same Popup with trail name. Per user decision: tapping polyline shows same name popup.
    - Set cursor to 'pointer' on mouseenter for unclustered/line-fill layers, reset on mouseleave.

    Export `routesToPointFeatures` and `pttkToLineFeatures` as named exports for potential testing.

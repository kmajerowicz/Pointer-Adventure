    4. **Cache check (PIPE-01):** Compute `bboxHash` (inline the function — cannot import from src/ in Deno). If `force` is true in the request body, skip the cache check (for the manual refresh feature). On cache hit, query `routes` within the bbox bounds (`.gte('center_lat', south).lte('center_lat', north).gte('center_lon', west).lte('center_lon', east)`) and return `{ routes, cached: true }`.

    5. **Overpass fetch (PIPE-02, PIPE-03, PIPE-06):** Build Overpass QL query (inline `buildOverpassQuery` logic). Parse JSON response. If all retries fail, return 502 JSON error `{ error: 'Overpass API niedostepny' }`.

    6. **Normalization (PIPE-04):** Inline the normalization functions (extractTrailColor, isPTTK, normalizeSurface, normalizeDifficulty, normalizeElement). Process each `element` in Overpass response `elements` array. Filter out null results. Set `water_access` to `'none'` by default (per research open question — skip around:200 subquery for v1).

Upsert normalized routes to `routes` table with `{ onConflict: 'source_id' }`. Insert new `search_areas` record with `bbox_hash`, bbox bounds, `fetched_at: new Date().toISOString()`, `expires_at` set to 7 days from now.

    8. **Response:** Return `{ routes: normalizedRoutes, cached: false }` with corsHeaders + Content-Type application/json.

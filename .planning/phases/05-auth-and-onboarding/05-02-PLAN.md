1. **Create src/features/auth/InvitePage.tsx:**
   - Read `token` from `useSearchParams`
   - If no token param: show "Dostep tylko przez zaproszenie" message (reuse for AUTH-04 pattern)
   - On mount, call `supabase.functions.invoke('validate-invite', { body: { token } })`
   - Show loading spinner during validation
   - If response `valid: false`:
     - `reason: 'expired'` or `'used'` → show "Zaproszenie wygaslo" + "Popros znajomego o nowe zaproszenie do Psi Szlak"
     - `reason: 'not_found'` → show "Nieprawidlowy link zaproszenia"
   - If response `valid: true`: render `<RegisterForm invitationId={data.invitation_id} token={token} />`
   - Use dark theme tokens: `bg-bg-base`, `text-text-primary`, `text-text-secondary`
   - Center content vertically, max-w-sm mx-auto, padding for mobile

2. **Create src/features/auth/RegisterForm.tsx:**
   - Props: `{ invitationId: string, token: string }`
   - Two fields: display name (text, required) and email (email, required)
   - "Dolacz do Psi Szlak" submit button (bg-accent, min-h-[48px] touch target)
   - On submit:
     a. Call `supabase.auth.signInWithOtp({ email, options: { shouldCreateUser: true, emailRedirectTo: window.location.origin, data: { display_name: name } } })`
     b. Only if signInWithOtp succeeds (no error): call `supabase.rpc('consume_invite', { p_token: token, p_user_id: null })` — actually, the user doesn't exist yet in public.users at this point. Instead, store the token in the MagicLinkSent state so it can be consumed after session is established.
     c. On success: switch to `<MagicLinkSent email={email} token={token} invitationId={invitationId} />`
     d. On error: show Polish error message below form
   - Per user decision: token consumed only AFTER magic link is successfully sent

3. **Create src/features/auth/MagicLinkSent.tsx:**
   - Props: `{ email: string, token: string, invitationId: string }`
   - Shows "Sprawdz swoj email" heading
   - Shows email address the link was sent to
   - **OTP input:** 6 separate single-char inputs with auto-focus advance (see RESEARCH.md code example). `inputMode="numeric"`, `maxLength={1}`. On all 6 digits entered, auto-submit via `supabase.auth.verifyOtp({ email, token: code, type: 'email' })`.
   - On successful verifyOtp: session is established (onAuthStateChange handles store update). Store the invite `token` in sessionStorage so App.tsx can consume it after profile is loaded.
   - **Resend link:** "Wyslij ponownie" button with 60-second countdown timer. Max 3 resends. After 3: show "Sprobuj pozniej". Use `useRef`-based interval for countdown (not setTimeout chain — avoids stale closure). Resend calls `supabase.auth.signInWithOtp({ email, options: { shouldCreateUser: true, emailRedirectTo: window.location.origin } })`.
   - All text in Polish. Dark theme tokens.
   - Show loading state during OTP verification.
   - On verifyOtp error: show "Nieprawidlowy kod. Sprobuj ponownie."

2. **Rewrite src/App.tsx** — This is the auth initialization point:
   - Import `useEffect` from React, `useNavigate` cannot be used outside Router. Since App renders `RouterProvider`, auth init must happen INSIDE the router. Create an `AuthInit` component rendered inside the router layout.
   - **Strategy:** Create an `AuthInit` wrapper component that wraps `RouterProvider` content. Actually, since `createBrowserRouter` is used, we need a different approach. Add `AuthInit` as a layout route element at the root that:
     a. Calls `supabase.auth.getSession()` on mount
     b. Sets session and profile in auth store
     c. Subscribes to `onAuthStateChange`
     d. Uses `useNavigate()` for redirects
   - **Implementation:** Modify `router.tsx` to wrap all routes in an `AuthInit` layout component. Or simpler: create a `useAuthInit` hook that runs in `AppLayout` and standalone pages.
   - **Simplest approach:** Create `src/hooks/useAuthInit.ts` that:
     a. On mount: `supabase.auth.getSession()` → `setSession(session)` → if session, fetch profile → `setProfile(profile)` → `setLoading(false)` → `setInitialized()`
     b. Subscribe to `onAuthStateChange((event, session) => { ... })` with `.then()` chains (NO await per RESEARCH.md pitfall 2)
     c. On `SIGNED_IN` event: fetch profile. If `!profile?.dog_name` → `navigate('/onboarding')`. Else → `navigate('/')`.
     d. On `SIGNED_OUT` event: `clear()` store.
     e. Check sessionStorage for pending invite token. If found and session exists: call `supabase.rpc('consume_invite', { p_token: token, p_user_id: session.user.id })` then remove from sessionStorage.
     f. Return `{ loading, initialized }` for rendering loading states
     g. CRITICAL per RESEARCH.md pitfall 4: Do NOT re-check dog_name from DB on every SIGNED_IN. Only redirect to onboarding on INITIAL_SESSION load or first SIGNED_IN after page had no session. Use a `hasRedirected` ref to prevent loops.
   - **Update App.tsx:** Call `useAuthInit()` at the top. Show a full-screen loading spinner (`bg-bg-base` with accent spinner) while `!initialized`. Once initialized, render `RouterProvider`.
   - Actually since `useNavigate` requires being inside a Router, and App.tsx creates the Router via `RouterProvider`, the auth init cannot use navigate. Better approach:
     - Keep App.tsx rendering `RouterProvider`
     - Create a root layout route component `AuthLayout` that wraps ALL routes
     - `AuthLayout` runs `useAuthInit` hook (which uses `useNavigate` internally)
     - `AuthLayout` renders `<Outlet />` once initialized, or loading spinner
